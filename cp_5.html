5  进程与系统状态

5.1 进程查看

5.1.1 ps：静态快照，而不是监控

ps 给你的不是“实时状态”，而是某一瞬间的进程快照。

这是它的优势，而不是缺点。

5.1.1.1 案例 1：看“我自己启动了什么”

ps

输出：

  PID TTY           TIME CMD
10270 ttys001    0:00.44 -zsh
79676 ttys003    0:00.14 /bin/zsh -il

你会看到当前终端关联的进程。

这一步的意义在于确认你现在这条 Shell， 自己派生出了哪些东西。

5.1.1.2 案例 2：看“我这个用户在跑什么”

ps -u $(whoami)

列出 你当前登录用户 运行的所有程序，包括终端、浏览器、微信等应用；后台服务、脚本；系统进程（以你身份运行的）

这在桌面系统上非常实用，不关心系统服务，只看“我自己的程序”。

5.1.1.3 案例 3：带点信息密度的常用写法

ps aux

你会看到用户、PID、CPU / 内存占用、启动命令。不要试图一次看懂所有列。你现在只需要知道一件事，ps 用来回答“是谁”， 不是“现在怎么样”。

5.1.2 top：正在发生什么

如果说 ps 是照片， 那 top 是直播画面。

5.1.2.1 案例 1：系统突然变慢时的第一反应

top

你立刻能看到哪个进程在吃 CPU、内存压力是否异常、系统是否真的“忙”。在 macOS 上，top 的一个重要用法是判断“卡顿是不是实时发生的”。如果 top 里一切平稳， 那问题很可能不是当前负载。

5.2 作业控制

这一节处理的是 Shell 内部的并发， 而不是系统级多任务。

5.2.1 &：把任务放到后台跑

sleep 60 &

你会立刻拿回提示符。这表示这个命令还在跑，但不再占着你的终端。这在桌面系统里非常常见：下载、简单脚本、临时任务。

5.2.2 jobs / fg / bg

jobs

你会看到当前 Shell 启动的后台任务。重要的是这个限定词：“这个 Shell”。

换一个终端窗口，你就看不到了。你可以先使用上述的sleep命令，再用这个jobs试试看。

fg

把最近的后台任务拉回前台。

bg

让暂停的任务继续在后台跑。

作业控制解决的是“我如何一边干别的，一边让这个命令继续跑。”

5.3 系统资源

这一节关注的不是“谁在跑”，而是系统整体还能不能扛住。

5.3.1 df / du

df -h

这是查看磁盘空间使用情况的命令，df 即disk free，磁盘空闲空间，-h 即human-readable，人类可读格式（自动用 GB/MB 显示）

这是任何异常排查的第一步之一。你关心的不是所有分区，而是：

  • /

  • 用户目录所在磁盘

du -h .

du 即disk usage，磁盘使用量，-h 即human-readable，人类可读（自动用 KB/MB/GB）

执行结果为：显示当前文件夹及其所有子文件/文件夹占用的磁盘空间。

你可以看到当前目录下哪些子目录占空间，占了多少。结合 sort 使用更清晰（BSD 原生也有）：

du -h . | sort -h

最后一行就是当前目录总大小。

这一步不是为了立刻删除，而是为了定位异常来源。

5.3.2 vm_stat：内存压力，而不是“内存用了多少”

macOS 的内存管理和 Linux 不同，“空闲内存”并不是一个好指标。

vm_stat

这是 查看 macOS 虚拟内存（Virtual Memory）状态 的命令。

vm 即virtual memory，虚拟内存，_stat 即status，状态统计。用于显示系统内存使用的底层统计数据。

你关注的不是具体数字，而是，是否频繁 page out、是否存在明显内存压力。

示例输出

Mach Virtual Memory Statistics: (page size of 16384 bytes)
Pages free:                              123456.
Pages active:                            234567.
Pages inactive:                          345678.
Pages wired down:                        12345.
Pages used by compressor:                56789.

注意：数值单位是 page（页），默认每页 16384 字节（16KB）

5.3.3 uptime：系统是否“长期疲劳”

uptime

这是 **查看系统运行时间和负载** 的命令。

你会看到系统运行了多久，最近一段时间的平均负载。

例如：16:30:00 up 3 days, 2:15, 2 users, load averages: 2.34 1.98 1.76
