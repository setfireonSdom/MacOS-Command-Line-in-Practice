6  网络与远程

在桌面系统里，网络不是一个抽象概念，而是一连串非常具体的问题：我现在有没有联网？是 Wi-Fi 的问题，还是对方的问题？这个接口到底有没有返回？我能不能直接把文件拉过来？

这一章不追求“理解网络协议”，而是让你在出现问题的那一刻，知道该用哪一个原生命令，把情况判断清楚。

6.1 网络基本状态

这一节解决的不是“网络怎么工作”，而是我现在这台机器，到底处在什么网络状态。

6.1.1 ping：最小可行的连通性判断

ping 只回答一个问题：“我能不能碰到对方？”

6.1.1.1 案例 1：确认网络是否通

ping baidu.com

如果你能看到持续返回：

64 bytes from ...

说明至少三件事成立，本地网络是通的、DNS 是通的、对方主机是可达的。

如果 ping 不通， 你就可以立刻停止往上层猜测。

6.1.1.2 案例 2：快速结束（桌面系统常用）

ping -c 4 google.com

只发 4 次包就退出，避免无休止刷屏。

6.1.2 ifconfig：我现在“接着”哪张网卡

ifconfig 不用来“配置”，而是用来确认状态。

ifconfig

你不需要看懂全部输出，只要抓住这几件事：

  • 哪个接口是 en0 / en1

  • 有没有 inet（IPv4 地址）

  • 是否是 status: active

现实判断是没有 IP， 后面所有网络问题都不用再查了。

6.1.3 netstat：连接是否真的建立

netstat 在桌面系统里，常用于回答一个非常具体的问题：

“这个连接到底有没有连上？”

6.1.3.1 案例 1：查看当前网络连接

netstat -an

这是查看网络连接和端口状态的命令，-a 即all，显示所有连接和监听端口，-n 即numeric，用数字显示地址和端口（不解析域名）

你可以看到本机正在建立的连接、监听的端口、连接状态（ESTABLISHED / LISTEN 等）

在调试网络程序、代理、端口占用时，这是比“感觉”可靠得多的判断方式。

6.1.3.2 常用组合

# 看谁在连你的 22 端口（SSH）
netstat -an | grep ".22 "

# 看所有 ESTABLISHED 连接
netstat -an | grep ESTABLISHED

# 统计连接数最多的 IP
netstat -an | awk '{print $5}' | cut -d: -f1 | sort | uniq -c | sort -rn

6.2 HTTP 与接口调试

6.2.1 curl（macOS 自带）

很多人第一次接触 curl，是用来“下文件”。但在真实工作中，它更常被用来验证接口是否存在、是否返回、返回了什么。

6.2.1.1 案例 1：最简单的 GET 请求

curl https://setfireonsdom.github.io/Ollama-Book/

你得到的不是“网页”，而是服务器返回的原始内容。这一步的价值在于跳过浏览器，直接面对服务器响应。

6.2.1.2 案例 2：只看响应头

curl -I https://setfireonsdom.github.io/Ollama-Book/

你可以立刻看到状态码（200 / 404 / 500）、重定向、内容类型。这是接口调试中非常高频的一步。

输出例子：

HTTP/2 200 
server: GitHub.com
content-type: text/html; charset=utf-8
......

6.2.1.3 案例 3：POST 请求（最小形态）

curl -X POST https://api.example.com/test

你不需要立刻写复杂参数，先确认这个接口，能不能被访问。

curl -X POST https://httpbin.org/post \
  -H "Content-Type: application/json" \
  -d '{"name":"张三","message":"Hello"}'

6.2.1.4 案例 4：带简单参数

curl "https://httpbin.org/get?q=test"

当你怀疑“接口逻辑”而不是“网络本身”时， curl 是最快的验证方式。

6.3 远程连接

这一节解决的不是“远程运维”，而是我如何直接连到另一台机器，或者把文件在两台机器之间移动。

6.3.1 ssh：远程登录的最低成本方式

ssh user@remote-host

成功之后，你得到的是另一台机器上的一个 Shell。这一步在现实中常用于访问远程服务器、登录公司内网机器、临时排查问题。

6.3.1.1 案例 1：确认“连不连得上”

ssh user@remote-host "echo ok"

如果你看到 ok，说明网络通、权限通、ssh 服务正常。不需要真正登录进去。

6.3.2 scp：最直接的文件传输

scp 解决的是一个非常朴素的问题，把这个文件，从这里，放到那里。

6.3.2.1 案例 1：本地 → 远程

scp report.txt user@remote-host:/home/user/

6.3.2.2 案例 2：远程 → 本地

scp user@remote-host:/home/user/report.txt .

你不需要任何额外工具，也不需要打开图形界面。

6.3.2.3 案例 3：传目录（递归）

scp -r data/ user@remote-host:/home/user/

这是桌面系统里非常常见的真实需求拷日志、拷结果、拷临时数据。
