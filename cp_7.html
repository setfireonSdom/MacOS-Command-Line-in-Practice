7  macOS 独有的原生命令

在前面的章节里，我们刻意只使用“任何 Unix 都可能有的能力”。从这一章开始，情况发生了变化。这里的命令不能直接照搬到 Linux，不是通用 Unix 教材的一部分，但在 macOS 上极其自然、极其高效。

7.1 open：连接 GUI 与 CLI

open 是 macOS 命令行里最被低估的命令之一。它的本质不是“打开文件”，而是把一个对象，交给系统默认的图形处理方式。

7.1.1 案例 1：用命令行打开文件（但用 GUI 看）

open data.csv

你没有指定任何程序， macOS 会自动用默认的 csv 阅读器打开。这意味着命令行负责“定位”，GUI 负责“展示”。

7.1.2 案例 2：打开目录（Finder 立刻介入）

open .

当前目录会直接在 Finder 中打开。这是一个非常常见的桌面工作流：在 CLI 里定位 → 用 Finder 做可视化操作。

7.1.3 案例 3：指定程序打开

open -a "WeChat.app" .

意思是启动微信 + 尝试打开当前文件夹，微信是聊天应用，不是文件编辑器，所以，微信启动，忽略 . 参数（微信不支持打开文件夹）

什么时候需要. ?

open -a "Visual Studio Code" .    ✅ 用 VS Code 打开当前项目
open -a "Sublime Text" .          ✅ 用 Sublime 打开当前文件夹
open -a "Finder" .                ✅ 用 Finder 打开当前目录（等于 open .）
open -a "WeChat" .                ❌ 微信不支持，等效于只启动

这一步不是“技巧”， 而是把 CLI 当作启动器。

7.1.4 案例 4：打开 URL

open https://setfireonsdom.github.io/Ollama-Book/

系统会自动用默认浏览器打开。这意味着你可以在脚本或命令中，触发真实的 GUI 行为。

7.2 pbcopy / pbpaste（剪贴板）

这一组命令解决的是一个非常具体的问题：如何让命令行， 直接参与“复制 / 粘贴”这个桌面动作。

7.2.1 pbcopy：把输出放进剪贴板

ls | pbcopy

它把ls的输出放进剪贴板，相当于复制了ls的输出。

现在你可以直接在任何 GUI 程序里粘贴。这一步的价值在于你不需要中间文件，也不需要手动选择文本。

cat data.csv| pbcopy

相当于复制data.csv的内容，这样不用打开文件，可以直接复制。

7.2.2 pbpaste：从剪贴板取内容

pbpaste

这会把当前剪贴板内容，当作标准输出打印出来。

7.2.2.1 案例 1：命令行 → GUI 的真实桥梁

grep error server.log | pbcopy

然后在邮件、文档、聊天工具里直接粘贴。这是真实工作中非常高频的用法。

7.2.2.2 案例 2：GUI → 命令行

你从网页或文档中复制了一段文本：

pbpaste | wc -l

wc -l 即word count -lines，统计行数。

你没有保存文件，但已经能在命令行里处理它。

7.3 say / afplay（系统能力）

命令行可以直接调用系统级能力。

7.3.1 say：把文本变成语音

say "Task completed"
say "你好"

这个真挺有意思的。macOS 会直接朗读这句话，现实用途包括：长任务完成提醒、脚本结束提示、无需盯屏幕的反馈。

7.3.1.1 案例 1：命令执行完成提示

sleep 5 && say "起床了"
cat data.csv | say

7.3.2 afplay：播放音频文件

afplay alert.wav

这是系统自带的音频播放接口。

7.3.2.1 案例 1：最简单的“声音通知”

echo "start" && afplay /System/Library/Sounds/Glass.aiff

afplay 即Audio File Play，macOS 播放音频文件。

/System/Library/Sounds/Glass.aiff 即系统自带音效路径，

7.4 log show（系统日志）

这是 macOS 命令行里最容易被误用、也最容易被低估的工具。macOS 的系统日志不是纯文本文件，不能直接 cat ，是统一日志系统（Unified Logging）

log show 是它的官方 CLI 接口。

7.4.1 案例 1：查看最近的系统日志

log show --last 1h

含义是：显示最近 1 小时的系统日志。你不需要理解所有输出，只需要知道这是系统“真实发生过的事”。

7.4.2 案例 2：按进程筛选

log show --predicate 'process == "kernel"' --last 10m

你可以把日志范围缩到一个明确对象。

7.4.3 案例 3：调试某个应用的问题

log show --predicate 'subsystem == "com.apple.network"' --last 30m

这是 Finder、系统设置无法直接做到的。log show 不是用来“翻日志”的， 而是用来“验证某件事是否真的发生过”。
