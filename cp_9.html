9  真实工作场景

9.1 场景一｜快速理解一个陌生目录

pwd
ls
ls -l
less test.txt
# 递归搜索当前目录下所有文件中的 "TODO" 标记，找出包含 TODO 
# 的所有行
grep -R TODO .

示例输出：

./src/main.py:15:    # TODO: 优化这段代码
./docs/readme.md:42:  TODO: 补充安装说明
./utils/helper.js:8:  // TODO 处理异常情况

9.2 场景二｜批量查看 / 修改文本

背景： 一堆日志 / 配置 / CSV 你不需要精修，只需要快速判断和批量处理。

# 只看关键行（而不是全文）
grep error server.log
# 如果太多
grep error server.log | less
# 排除干扰信息
grep error server.log | grep -v DEBUG

这是现实中极其常见的二次筛选。

# 批量预览替换效果（不改文件）
sed 's/http/https/' config.conf

先看输出，再决定是否改文件。

# 真正修改（macOS 写法）
sed -i '' 's/http/https/' *.conf

这一步之前，你应该已经确认过：

ls *.conf

9.3 场景三｜清理磁盘空间

背景： 系统开始变慢、下载失败、莫名其妙报错 第一怀疑对象永远是：磁盘

# 先看“是不是整体问题”
df -h
# 再看“是谁在占”
du -h .
du -h . | sort -h
# 定位“大而异常”的文件
find . -size +500M

9.4 场景四｜定位异常进程

背景： 风扇狂转 / 电池狂掉 / 系统卡顿 你不想重启，你想知道原因。

# 先看“是谁在跑”
top
再确认“这是谁启动的”
ps aux

9.5 场景五｜查系统日志定位问题

背景： 程序异常、系统行为反常 Finder 看不出来，你需要事实。

# 先限定时间范围
log show --last 10m
# 按进程缩小范围
log show --predicate 'process == "kernel"' --last 10m
# 或者针对具体应用：
log show --predicate 'process == "MyApp"' --last 10m
