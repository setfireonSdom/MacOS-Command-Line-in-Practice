3  文本处理

日志、CSV、配置、代码、导出的数据——它们的共同点是，都是文本。

你不需要数据库， 不需要 Excel， 甚至不需要写脚本。你只需要学会一件事，让文本从一行行字符，变成可以被筛选、变形、组合的材料。

3.1 grep（原生版）

grep 做的事情只有一件，从文本中，把“符合条件的行”挑出来。

假设你有一个日志文件：

server.log

第一反应永远不是打开编辑器，而是：

grep error server.log

这个指令是 获取包含 “error” 的行（不是以 error 开头）。

3.1.1 常用但足够的模式

你不需要记正则大全，只要三种：

# 1 忽略大小写
grep -i error server.log

适合日志、用户输入、混乱来源。

# 2 反向匹配（排除）
grep -v error server.log

不是“找什么”，而是“不要什么”。

# 3 递归搜索目录
grep -R TODO .

这是在代码仓库里快速摸底的常用方式。

3.1.2 快速确认“有没有发生过”

grep "connection refused" server.log

如果没有输出，你就可以立即得出一个结论：至少从文本记录上看，没有发生过。这比“我感觉应该没有”要可靠得多。

3.2 sed（macOS 特有写法）

sed 的角色是在文本通过时，对内容进行规则化修改。它不是编辑器，它是文本变换器。

3.2.1 预览式替换（不改文件）

sed 's/error/ERROR/' server.log

这是 用 sed 命令替换文件中的文本，读取 server.log 的每一行，每行第一个出现的 error 替换成 ERROR 。输出到屏幕（不修改原文件）

你会看到输出变了， 但文件本身没有被修改。 这是 sed 的安全使用方式。

3.2.2 -i ’’

在 macOS（BSD sed）里，就地修改文件必须这样写：

sed -i '' 's/error/ERROR/' server.log

-i ''：表示不生成备份文件。

少了这个空字符串，命令会直接失败。这是 macOS 用户必须知道的一个事实差异。

3.2.3 批量修正配置

你有一批文件：

ls *.conf

你要统一替换一个值：

sed -i '' 's/localhost/127.0.0.1/g' *.conf

g代表所有匹配的内容，没有它就只匹配每一行碰到的第一个。

先确认 glob 展开的对象， 再执行 sed。

3.3 awk（字段级武器）

如果说 grep 是“行筛选”， sed 是“行变形”， 那 awk 是，在一行内部，按字段进行计算和重组。

3.3.1 最小可用集

  • $1, $2, $3：第 1、2、3 列

  • $0：整行

  • -F：字段分隔符

3.3.2 实战案例

3.3.2.1 1 从 CSV 中取你真正要的列

假设文件 data.csv 内容是：

id,name,score
1,Alice,87
2,Bob,92
3,Charlie,78

你只关心名字和分数：

awk -F',' '{print $2, $3}' data.csv

输出立刻变成：

name score
Alice 87
Bob 92
Charlie 78

3.3.2.2 2 带条件的筛选

只看分数大于 85 的：

awk -F',' '$3 > 85 {print $2, $3}' data.csv

你已经在不写脚本的情况下做判断了。

name score
Alice 87
Bob 92

3.3.2.3 3 简单统计

awk -F',' '{sum += $3} END {print sum}' data.csv

这已经是“计算”，而不仅是查看。

3.4 管道与重定向

这一节不是新工具，而是把前面所有工具连起来的方式。

3.4.1 |：把一个命令的输出，变成下一个的输入

grep error server.log | less

这是现实世界里最常见的组合之一：

  • grep：缩小范围

  • less：可控查看

3.4.1.1 案例

grep error -i server.log | grep -v Error

你不是一次写完条件，而是逐层逼近目标。

3.4.2 > 与 >>：输出去向

grep error server.log > test.txt

这是覆盖写入。

grep error server.log >> test.txt

这是追加写入。区别不在符号，而在后果。

3.4.3 <：输入重定向（较少见，但存在）

awk '{print $1}' < data.txt

明确告诉命令：输入来自文件，而不是键盘。

3.4.3.1 组合思维（完整流程）

你拿到一个巨大日志文件，想做三件事：

  1. 只看 error
  2. 排除 DEBUG
  3. 保存结果
grep error server.log \
  | grep -v DEBUG \
  > clean_errors.log
